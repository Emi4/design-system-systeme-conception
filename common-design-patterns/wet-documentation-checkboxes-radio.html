Checkboxes and radio buttons - Documentation

Purpose

Checkboxes and radio buttons with large touch targets to support behavioural patterns of interacting with the form element, and not the label.

Usage

Use when

  • Steamline complex navigation through multiple option.
  • Use for interactive questions and checkboxes for services and wizards

Do not use when

  • Avoid using checkboxes or radio buttons if the options are not easily comparable. Consider navigation patterns such as a topic page
  • If supporting navigation, for a small number of options, consider using expand/collapse or tabs

Working examples

Checkboxes

Appearance
Large checkboxes
Small checkboxes
Correct use

Compliance point(s):

  • Use large checkboxes when one or more selections can be made to answer a question. It can also be used for a checklist, with short labels that do not break across lines.
  • Use small checkboxes when making a checklist with items that would break across multiple lines or when there is a single item that can be checked.
Incorrect use

Compliance point(s):

  • Do not use if there is a need for users to enter a detailed answer
  • Do not use the small checkbox pattern in wizards or questions in forms

Radio buttons
Correct use

Compliance point(s):

  • Use radio buttons when only one selection can be made.
Incorrect use

Compliance point(s):

  • Do not use if there is a need for users to select more than one option
