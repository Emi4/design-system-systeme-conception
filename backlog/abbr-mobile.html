ABBR style for mobile/touch devices

In mobile, ABBR tags do not repond to touch without adding extra hover. However, the hover only lasts as long as the user presses the element and leaves (in desktop), or moves to the next focused type item (in mobile).

Below are possible solutions for mobile touch with ABBR elements.

Default show of title value in mobile (CSS only)

This example, the title attribute is always shown as an :after pseudo element in small/tablet modes (most used with touch). The text-decoration is removed since screen readers can still pick up on the ABBR tag but visual users will not need them to determine that something is an ABBR since it's defined in the CSS.

The CRA is working with TBS to help enhance the WET5 template.

Pros:

  • Simplest to impliment
  • Acts just like regular text

Cons:

  • May not be desirable where width space is a premium (e.g. table header)

Use :hover and few other CSS styles to create an almost "click" expand (CSS only)

This example, the title attribute is activated with :hover:after pseudo element in small/tablet modes (most used with touch) are pressed. The text-decoration is replaced with a similar border-bottom pattern so that when active, the border disapears (and does not cascade into the :after element).

This design uses animation and holds the after effect for 180 seconds (can be adjusted) so the item does not disappear right after the user moves away (by accident or on purpose).

The CRA is working with TBS to help enhance the WET5 template.

Pros:

  • Does not provide abbrs unless user clicks
  • Acts just like regular text

Cons:

  • May be mistaken as an expand hide (does not dissapear for 3 mins)

Use script

Using script, could create an expand/hide, popup (e.g. popover) or any other type of function (depending on best experience).
